## ساختار کلی

فایل `content.json` دارای ۵ بخش اصلی است:

```json
{
  "company": { ... },           // اطلاعات شرکت
  "reply_keyboard": { ... },    // دکمه‌های پایین صفحه
  "screens": { ... },           // ✨ صفحات اصلی
  "products": [ ... ],          // لیست محصولات
  "pricing": [ ... ]            // لیست پلن‌های قیمتی
}
```

---

## بخش company

اطلاعات شرکت شما. این داده‌ها در صفحه `contact` جایگزین می‌شوند.

```json
{
  "company": {
    "name": "تراز",
    "phone": "+982191000000",
    "email": "support@taraz.example.com",
    "bale_username": "@taraztest_bot",
    "website": "https://taraz.example.com"
  }
}
```

### توضیح فیلدها:
- `name`: نام شرکت (درون متن‌های صفحات استفاده می‌شود)
- `phone`: شماره تلفن (صفحه تماس)
- `email`: آدرس ایمیل (صفحه تماس)
- `bale_username`: نام کاربری Bale بات شما
- `website`: وب‌سایت شرکت

## بخش screens

**این جا تمام صفحات و منوها تعریف می‌شوند.**

### ساختار یک صفحه:

```json
{
  "screens": {
    "screen_key": {
      "text": "محتوای صفحه",
      "parent": "منو_والد",           // اختیاری (برای دکمه Back)
      "buttons": [...]               // اختیاری (دکمه‌ها)
    }
  }
}
```

### فیلدهای صفحه:

| فیلد | ضروری | توضیح |
|------|------|--------|
| `text` | ✅ | محتوای صفحه (از متغیرهای `{...}` پشتیبانی می‌کند) |
| `parent` | ❌ | نام صفحه والد (برای دکمه Back خودکار) |
| `buttons` | ❌ | آرایه ۲بعدی دکمه‌ها |

### مثال - صفحه ساده:

```json
{
  "screens": {
    "about": {
      "text": "🏢 درباره ما\n\nما یک شرکت نرم‌افزاری ERP هستیم.",
      "parent": "main_menu",
      "buttons": [
        [
          {"text": "🌐 وب‌سایت", "url": "https://example.com"}
        ],
        [
          {"text": "🏠 منوی اصلی", "route": "main_menu"}
        ]
      ]
    }
  }
}
```

### صفحات ویژه (خودکار):

سه صفحه **خودکار** توسط کد پردازش می‌شوند:

```json
{
  "product_detail": {
    "text": "این صفحه محصول به صورت داینامیک توسط سیستم پر می‌شود.",
    "parent": "products_list"
  },
  
  "pricing_detail": {
    "text": "این صفحه قیمت به صورت داینامیک توسط سیستم پر می‌شود.",
    "parent": "pricing_list"
  },
  
  "demo_for_product": {
    "text": "این صفحه دمو محصول به صورت داینامیک توسط سیستم پر می‌شود.",
    "parent": "product_detail"
  }
}
```

**نکته:** این صفحات نیاز به ویرایش ندارند!

## بخش products

لیست محصولات شرکت.

### ساختار محصول:

```json
{
  "products": [
    {
      "key": "crm",                    // کلید یکتا
      "short_title": "مدیریت ارتباط", // عنوان کوتاه (در لیست)
      "title": "📇 تراز CRM",          // عنوان کامل
      "tagline": "هر مشتری را بشناسید", // شعار
      "text": "توضیح مفصل محصول...",    // متن صفحه جزئیات
      "image_path": "assets/crm_promo.png",  // ❌ عکس تبلیغی (اختیاری)
      "pdf_path": "assets/crm_features.pdf", // ❌ PDF (اختیاری)
      "buttons": [
        [{"text": "🚀 درخواست دمو", "route": "demo_for_product", "arg": "crm"}],
        [{"text": "🔙 بازگشت", "route": "products_list"}],
        [{"text": "🏠 منوی اصلی", "route": "main_menu"}]
      ]
    }
  ]
}
```

### توضیح فیلدها:

- `key`: کلید یکتا برای محصول (بدون فاصله، بدون مشخصه خاص)
- `short_title`: نمایش شده در لیست محصولات
- `title`: نمایش شده در صفحه جزئیات
- `tagline`: شعار بازاریابی
- `text`: توضیح مفصل (از متغیرها پشتیبانی می‌کند)
- `image_path`: مسیر عکس (اختیاری)
- `pdf_path`: مسیر PDF (اختیاری)
- `buttons`: دکمه‌های صفحه جزئیات

### نمونه محصول ساده:

```json
{
  "key": "task_management",
  "short_title": "مدیریت وظایف",
  "title": "🗂 مدیریت وظایف و پروژه",
  "tagline": "برنامه‌ریزی کنید، پیگیری کنید، تحویل دهید",
  "text": "🗂 مدیریت وظایف و پروژه\n\n✨ ویژگی‌ها:\n• برد کانبان\n• نمودار گانت\n• ردیابی زمان",
  "buttons": [
    [{"text": "🚀 درخواست دمو", "route": "demo_for_product", "arg": "task_management"}],
    [{"text": "🔙 بازگشت", "route": "products_list"}],
    [{"text": "🏠 منوی اصلی", "route": "main_menu"}]
  ]
}
```

## ساختار دکمه‌ها

دکمه‌ها در دو جای استفاده می‌شوند:
1. **صفحات** (`screens[key].buttons`)
2. **منوی ریپلای** (`reply_keyboard.buttons`)

### ساختار دکمه:

```json
{
  "text": "📞 تماس",              // ✅ ضروری - متن دکمه
  "route": "contact",            // ❌ یا `route` یا `url` یا `action`
  "url": "https://example.com",  // ❌ لینک خارجی
  "action": "main_menu",         // ❌ اکشن خاص
  "arg": "product_key"           // ❌ اختیاری - پارامتر
}
```

### انواع دکمه‌ها:

#### ۱. دکمه Route (مسیر)

```json
{
  "text": "📦 محصولات",
  "route": "products_list"    // برای صفحه‌ای در `screens`
}
```

#### ۲. دکمه URL (لینک خارجی)

```json
{
  "text": "🌐 وب‌سایت",
  "url": "https://example.com"
}
```

#### ۳. دکمه Action (اکشن خاص - فقط ریپلای کیبورد)

```json
{
  "text": "🏠 منوی اصلی",
  "action": "main_menu"
}
```

#### ۴. دکمه با پارامتر (Arg)

```json
{
  "text": "🚀 درخواست دمو",
  "route": "demo_for_product",
  "arg": "crm"    // نام محصول
}
```

### ترتیب دکمه‌ها:

دکمه‌ها در **آرایه‌های دو بعدی** تعریف می‌شوند:

```json
{
  "buttons": [
    [                              // سطر ۱
      {"text": "دکمه ۱", ...},
      {"text": "دکمه ۲", ...}      // دو دکمه کنار هم
    ],
    [                              // سطر ۲
      {"text": "دکمه ۳", ...}      // یک دکمه تنهایی
    ]
  ]
}
```

## متغیرهای خودکار

برخی فیلدها از **متغیرهای خودکار** پشتیبانی می‌کنند:

### متغیرهای موجود:

```json
{
  "text": "سلام! 👋 به {company_name} خوش آمدید.\n\n📞 تماس: {phone}"
}
```

**متغیرهای دستیاب:**
- `{company_name}` → نام شرکت
- `{phone}` → شماره تلفن
- `{email}` → ایمیل
- `{bale_username}` → نام کاربری Bale
- `{website}` → وب‌سایت

### مثال - صفحه تماس:

```json
{
  "screens": {
    "contact": {
      "text": "📞 تماس با {company_name}\n\n☎️ {phone}\n✉️ {email}\n💬 {bale_username}",
      "buttons": [
        [{"text": "🏠 منوی اصلی", "route": "main_menu"}]
      ]
    }
  }
}
```

## ✅ Checklist - قبل از ارسال

- [ ] تمام `"key"` ها بدون فاصله و مشخصه خاص هستند
- [ ] تمام `"route"`ها به `screen_key` موجود اشاره می‌کنند
- [ ] تمام متغیرها `{...}` درست نوشته شده‌اند
- [ ] دکمه‌های Back `"parent"` دارند
- [ ] Emojis درست انتخاب شده‌اند
- [ ] JSON syntax معتبر است
