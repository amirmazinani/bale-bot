# راهنمای سیستم کاملاً داینامیک

## 🎯 معرفی
حالا بات شما **کاملاً داینامیک** شده! میتونید:

1. **صفحات جدید** اضافه کنید
2. **دکمههای جدید** اضافه کنید  
3. **منوها** رو تغییر بدید
4. **مسیرها** رو تغییر بدید

**بدون نیاز به ویرایش هیچ فایل کدی!**

## 📁 ساختار جدید content.json

### ۱. بخش screens (صفحات)
```json
"screens": {
  "screen_key": {
    "text": "متن صفحه",
    "parent": "parent_screen_key",
    "buttons": [...]
  }
}
```

### ۲. ویژگیهای صفحه:
- **`screen_key`**: کلید یکتا برای صفحه (مثل `blog`, `special_products`)
- **`text`**: متن صفحه
- **`parent`**: صفحه والد (برای دکمه Back خودکار)
- **`buttons`**: آرایه دکمهها (اختیاری)

### ۳. ساختار دکمه:
```json
{
  "text": "متن دکمه",
  "route": "screen_key",        // رفتن به صفحه دیگر
  "action": "action_name",      // اکشن خاص
  "url": "https://...",         // لینک خارجی
  "arg": "value"               // پارامتر اختیاری
}
```

## 🚀 مثال: اضافه کردن صفحه جدید

### قدم ۱: اضافه کردن صفحه در JSON
```json
"screens": {
  "our_team": {
    "text": "👥 تیم ما\\n\\nاعضای تیم حرفهای ما...",
    "parent": "main_menu",
    "buttons": [
      [
        {
          "text": "📞 تماس",
          "route": "contact"
        }
      ],
      [
        {
          "text": "🏠 منوی اصلی",
          "route": "main_menu"
        }
      ]
    ]
  }
}
```

### قدم ۲: اضافه کردن دکمه در منوی اصلی
```json
"main_menu": {
  "buttons": [
    // ... دکمههای قبلی
    [
      {
        "text": "👥 تیم ما",
        "route": "our_team"  // همین!
      }
    ]
  ]
}
```

### قدم ۳: اضافه کردن به ریپلای کیبورد (اختیاری)
```json
"reply_keyboard": {
  "buttons": [
    // ... دکمههای قبلی
    {
      "text": "👥 تیم",
      "route": "our_team"
    }
  ]
}
```

### قدم ۴: ریاستارت بات
```bash
# بات رو ریاستارت کنید
python main.py
```

**تمام!** صفحه جدید با همه دکمهها و مسیرها به صورت خودکار ایجاد میشه.

## 🔧 پشتیبانی از اکشنهای خاص

### ۱. اکشنهای فعلی:
```json
{
  "text": "📞 تماس",
  "action": "contact"  // پشتیبانی شده
}
```

```json
{
  "text": "🏠 منوی اصلی", 
  "action": "main_menu"  // پشتیبانی شده
}
```

### ۲. استفاده از route (ترجیح داده شده):
```json
{
  "text": "📝 وبلاگ",
  "route": "blog"  // بهتر از action هست
}
```

## 🌐 ساختار مسیرهای خودکار

هر صفحه به صورت خودکار مسیر دریافت میکنه:

```
screen_key    ->   menu:screen_key
blog          ->   menu:blog
about         ->   menu:about  
contact       ->   menu:contact
```

## 📊 مثال کامل: سیستم FAQ

### ۱. اضافه کردن صفحات FAQ:
```json
"screens": {
  "faq_main": {
    "text": "❓ سوالات متداول\\n\\nموضوع مورد نظر را انتخاب کنید:",
    "parent": "main_menu",
    "buttons": [
      [
        {
          "text": "💰 سوالات قیمت",
          "route": "faq_pricing"
        }
      ],
      [
        {
          "text": "📦 سوالات محصول",
          "route": "faq_products"
        }
      ],
      [
        {
          "text": "🏠 منوی اصلی",
          "route": "main_menu"
        }
      ]
    ]
  },
  
  "faq_pricing": {
    "text": "💰 سوالات متداول قیمت\\n\\n۱. قیمتها چطور محاسبه میشه؟\\n۲. امکان پرداخت اقساطی وجود داره؟\\n۳. تست رایگان چند روزه؟",
    "parent": "faq_main",
    "buttons": [
      [
        {
          "text": "📞 مشاوره رایگان",
          "route": "contact"
        }
      ],
      [
        {
          "text": "🔙 بازگشت",
          "route": "faq_main"
        }
      ]
    ]
  }
}
```

### ۲. اضافه کردن دکمه FAQ به منوی اصلی:
```json
"main_menu": {
  "buttons": [
    // ... دکمههای دیگر
    [
      {
        "text": "❓ سوالات متداول",
        "route": "faq_main"
      }
    ]
  ]
}
```

## ⚡ نکات فنی

### ۱. parent فیلد:
- **`null`**: صفحه ریشه (مثل `main_menu`)
- **`"main_menu"`**: برگشت به منوی اصلی
- **`"screen_key"`**: برگشت به صفحه خاص

### ۲. اکشنهای پشتیبانی شده در ریپلای کیبورد:
- `"action": "main_menu"`
- `"action": "contact"`

### ۳. URL در دکمهها:
```json
{
  "text": "🌐 وبسایت",
  "url": "https://example.com"
}
```

### ۴. متغیرها در متن:
```json
{
  "text": "تماس با {company_name}",
  // به صورت خودکار جایگزین میشه
}
```

## 🧪 تست سیستم داینامیک

### قدم ۱: کپی کردن فایل داینامیک:
```bash
copy content\\content.dynamic.json content.json
```

### قدم ۲: ریاستارت بات:
```bash
python main.py
```

### قدم ۳: تست:
1. منوی اصلی رو باز کنید
2. دکمه "⭐ ویژه" رو بزنید
3. دکمه "📝 وبلاگ" رو بزنید
4. دکمههای ریپلای کیبورد رو تست کنید

## 🔍 عیبیابی

### مشکل: صفحه نمایش داده نمیشه
**علت**: صفحه در `screens` تعریف نشده
**راهحل**: مطمئن شید `screen_key` درست نوشته شده

### مشکل: دکمه کار نمیکنه
**علت**: `route` اشتباه نوشته شده
**راهحل**: از `screen_key` موجود استفاده کنید

### مشکل: دکمه Back کار نمیکنه
**علت**: `parent` تعریف نشده
**راهحل**: `parent` رو اضافه کنید

## 📈 مزایای سیستم داینامیک

1. **بدون کدنویسی**: فقط JSON ویرایش کنید
2. **تست سریع**: تغییرات فوری اعمال میشن
3. **امن**: خطای کدنویسی نداریم
4. **قابل توسعه**: صفحات نامحدود اضافه کنید
5. **مدیریت آسان**: همه چیز در یک فایل

## 🎮 شروع سریع

برای اضافه کردن صفحه جدید:

1. صفحه رو به `screens` اضافه کنید
2. دکمه رو به `main_menu.buttons` اضافه کنید  
3. بات رو ریاستارت کنید
4. تست کنید!
