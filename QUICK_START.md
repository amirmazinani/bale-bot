# 🚀 شروع سریع - سیستم کاملاً داینامیک

## 📝 نحوه استفاده

### ۱. ویرایش صفحات و دکمهها
فقط فایل `content.json` رو ویرایش کنید:

```json
{
  "screens": {
    "new_page": {
      "text": "متن صفحه جدید...",
      "parent": "main_menu",
      "buttons": [
        [
          {
            "text": "🏠 منوی اصلی",
            "route": "main_menu"
          }
        ]
      ]
    }
  }
}
```

### ۲. اضافه کردن دکمه به منوی اصلی
```json
{
  "main_menu": {
    "buttons": [
      // ... دکمههای قبلی
      [
        {
          "text": "📝 صفحه جدید",
          "route": "new_page"
        }
      ]
    ]
  }
}
```

### ۳. ریاستارت بات
```bash
python main.py
```

## 🎯 مثال: اضافه کردن صفحه FAQ

### ۱. صفحه FAQ رو اضافه کنید:
```json
{
  "screens": {
    "faq": {
      "text": "❓ سوالات متداول...",
      "parent": "main_menu",
      "buttons": [
        [
          {
            "text": "📞 تماس",
            "route": "contact"
          }
        ],
        [
          {
            "text": "🏠 منوی اصلی",
            "route": "main_menu"
          }
        ]
      ]
    }
  }
}
```

### ۲. دکمه FAQ به منوی اصلی:
```json
{
  "text": "❓ سوالات",
  "route": "faq"
}
```

### ۳. ریاستارت کنید و تست کنید!

## 📁 ساختار فایل content.json

```
{
  "company": {...},          // اطلاعات شرکت
  "reply_keyboard": {...},   // دکمههای پایین صفحه
  "screens": {...},          // ✨ همه صفحات اینجا تعریف میشن
  "products": [...],         // لیست محصولات
  "pricing": [...]           // لیست قیمتها
}
```

## ⚡ ویژگیهای سیستم داینامیک

- **بدون کدنویسی** - فقط JSON
- **تغییرات فوری** - ریاستارت کنید
- **مسیرهای خودکار** - هر صفحه: `menu:screen_key`
- **دکمه Back خودکار** - از parent فیلد استفاده میکنه

## 🔧 عیبیابی سریع

### مشکل: صفحه نمایش داده نمیشه
✅ مطمئن شید `screen_key` در `screens` تعریف شده

### مشکل: دکمه کار نمیکنه  
✅ مطمئن شید `route` درست نوشته شده

### مشکل: بات اجرا نمیشه
✅ JSON syntax رو چک کنید
